# 第三天

## Servlet urlPattern配置

| 匹配类型   | 语法        | 匹配逻辑                   | 优先级 | 注意事项                                                     |
| ---------- | ----------- | -------------------------- | ------ | ------------------------------------------------------------ |
| 精确匹配   | `/xxx/xxx`  | 完全一致才匹配             | 最高   | 优先级最高，优先匹配                                         |
| 目录匹配   | `/xxx/*`    | 匹配该目录下所有子路径     | 次高   | `*` 只能在末尾，不能 `/user/*/aaa`                           |
| 扩展名匹配 | `*.do`      | 匹配以该后缀结尾的所有路径 | 中等   | 不能加 `/` 开头，如 `/ *.do` 是错误写法                      |
| 任意匹配   | `/` 或 `/*` | 匹配所有未匹配到的路径     | 最低   | `/` 会覆盖 Tomcat 默认 Servlet，`/*` 会拦截所有请求（包括静态资源） |

3. Request请求转发

**请求转发 (forward)**：一种在服务器内部的资源跳转方式

req.getRequestDispatcher("资源B路径").forward(req,resp);



**重定向 (Redirect)**：一种资源跳转方式

resp.setStatus(302); resp.setHeader("location","资源B的路径"); 

resp.sendRedirect("资源B的路径");

| 对比项   | 请求转发（Forward）                    | 重定向（Redirect）                     |
| -------- | -------------------------------------- | -------------------------------------- |
| 方法     | `req.getRequestDispatcher().forward()` | `resp.sendRedirect()`                  |
| 请求次数 | 1 次（服务器内部转发）                 | 2 次（浏览器重新发起请求）             |
| 地址栏   | 不变                                   | 变为新的目标地址                       |
| 数据共享 | 共享同一个 request 对象                | 不共享（两次独立请求）                 |
| 路径限制 | 只能访问本项目内资源                   | 可以访问任意外部资源                   |
| 效率     | 高（一次请求）                         | 低（两次请求）                         |
| 典型场景 | 表单提交后转发到结果页、共享数据       | 登录成功后跳转到首页、防止表单重复提交 |

​       **Request 域对象的 3 个核心方法**

- `void setAttribute(String name, Object o)`：存储数据到 request 域中
- `Object getAttribute(String name)`：根据 key，获取值
- `void removeAttribute(String name)`：根据 key，删除该键值对

 **3.1. 响应行（Response Line）**

示例响应行

```
HTTP/1.1 200 OK
```

- 组成：`HTTP版本 状态码 状态描述`
- 核心：**状态码** 用来告诉浏览器本次请求的处理结果（如 200 成功、404 未找到、302 重定向、500 服务器错误）

对应 API

`void setStatus(int sc)`：设置响应状态码

- 常用状态码：

  - `200`：请求成功（默认）
  - `302`：重定向
  - `404`：请求资源不存在
  - `500`：服务器内部错误

  

- 示例：`resp.setStatus(404);` 手动设置 404 状态码

------

**3.2 响应头（Response Headers)**

 示例响应头

```
Content-Type: text/html
```

- 作用：用键值对的形式，告诉浏览器响应的附加信息（如数据类型、编码、缓存策略等）

**对应 API**

`void setHeader(String name, String value)`：设置响应头键值对

- 常用响应头：

  - `Content-Type`：指定响应体的数据类型（如 `text/html` 表示 HTML、`application/json` 表示 JSON），同时可指定编码（`text/html;charset=UTF-8`，解决响应中文乱码）
  - `Location`：配合 302 状态码，实现重定向
  - `Content-Disposition`：用于文件下载，指定下载文件名

  

- 示例：`resp.setHeader("Content-Type", "text/html;charset=UTF-8");` 解决响应中文乱码

------

**3.3 响应体（Response Body）**

示例响应体

```
<html><head>head<body></body></html>
```

- 作用：**真正返回给浏览器的内容**（HTML、JSON、文件字节等），是用户最终看到的页面 / 数据

对应 API（2 个输出流，二选一，不可同时使用）

| 方法                                    | 作用               | 适用场景                                                |
| --------------------------------------- | ------------------ | ------------------------------------------------------- |
| `PrintWriter getWriter()`               | 获取**字符输出流** | 输出文本数据（HTML、普通字符串、JSON 等），开发中最常用 |
| `ServletOutputStream getOutputStream()` | 获取**字节输出流** | 输出二进制数据（图片、文件下载、视频等）                |
